You are helping develop SENTINEL, a self-hosted OSINT and cyber operations dashboard.

Project overview:
SENTINEL is an all-in-one OSINT platform for cyber operations. It supports threat intelligence, dark web monitoring, cyber surface monitoring, GEOINT, SOCMINT, news aggregation, profile intelligence, alerts, reports, and user/admin management. It is designed for Docker Compose deployment on low-resource infrastructure using mostly free/open data sources.

Tech stack:
- Backend: Python 3.11, FastAPI, async SQLAlchemy 2.0, Pydantic v2, Alembic
- Frontend: Next.js 14 App Router, React 18, TypeScript, Tailwind CSS
- Database: PostgreSQL 16
- Cache/queue: Redis 7
- Background jobs: Celery + Redbeat
- Auth: JWT access/refresh tokens, bcrypt, RBAC with admin/analyst/viewer roles
- Real-time: FastAPI WebSocket at `/ws/{channel}?token=<jwt>` using Redis pub/sub
- Deployment: Docker Compose with Nginx reverse proxy

Important API areas:
- Auth:
  - `POST /api/auth/login`
  - `POST /api/auth/refresh`
  - `GET /api/auth/me`
- Threat intelligence:
  - `POST /api/threat-intel/lookup`
  - IOC history, feeds, bulk lookup, Shodan helpers, stats
- Dark web:
  - Legacy route group: `/api/dark-web`
  - Expanded route group: `/api/darkweb`
  - Supports keywords, mentions, breach lookup, ransomware victims, scans, forum mentions
- Cyber surface:
  - `/api/cyber-surface/assets`
  - scans, vulnerabilities, stats
- News:
  - `/api/news/articles`
  - sources, keywords, fetch-now, score-now, alerts, stats
- Alerts:
  - `/api/alerts`
  - acknowledge, resolve, assign, reports, notification configs

Background jobs:
Celery is configured in `backend/app/core/celery_app.py`.
Recurring jobs include:
- threat feed fetching
- stale IOC refresh
- dark web watchlist scans
- news fetching and relevance scoring
- cyber surface scans
- alert processing
- data archiving
- ransomware/live, RSS, paste, search, and forum darkweb scans
